Извлеки из конспекта атомарные knowledge units, пригодные для генерации учебных flashcards.

Требования:
- Верни не более {max_units} knowledge units.
- Одна knowledge unit = одна мысль.
- Не возвращай raw chunks текста и обрывки фраз.
- Не объединяй несколько независимых фактов в один unit.
- Каждый unit должен быть завершённым по смыслу.
- Для каждого unit нормализуй topic: коротко, осмысленно, без случайных кусков предложения.
- Если во фрагменте есть определение, topic должен быть термином, а не смесью слов вроде `эукариоты определение эукариоты организмы`.
- Если во фрагменте есть сравнение, topic должен обозначать сравниваемые объекты, а не слово `сравнение` или подпись `признак`.
- Если comparison fragment содержит строки или ячейки вида `Признак A B`, извлекай структурированные units:
  - summary/list unit по критериям сравнения;
  - relation units по отдельным признакам, если признак и значения можно восстановить.
- Если list block состоит из коротких пунктов под общим заголовком, собирай его в один `list_item`, а не в сырой набор отдельных обрывков.
- Если видишь пример, помечай его как `example`, а не как основной факт.
- Если видишь перечисление или состав, используй `list_item`.
- Если видишь этап процесса, используй `process_step`.

Допустимые значения `type`:
- `definition`
- `concept`
- `fact`
- `relation`
- `example`
- `list_item`
- `process_step`

Формат ответа:
- Верни ТОЛЬКО валидный JSON-массив.
- Каждый элемент массива должен иметь поля:
  - `type`
  - `topic`
  - `content`
  - `source_fragment`
  - `importance`
  - `confidence`
